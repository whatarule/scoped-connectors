# スキル分割計画

## Context

現在は単一スキル `/slack` でサブコマンド分岐しているが、
SKILL.md が長くなり、サブコマンドの補完も効かない。

スクリプトに全責任を移す（curl 廃止計画）と SKILL.md はスクリプト呼び出しだけになるので、
分割しても各 SKILL.md は数行で済む。スクリプトは共有ディレクトリに置く。

## 方針

単一スキル `/slack` を5つのスキルに分割する。
スクリプトは共有ディレクトリ `skills/slack/scripts/` に置き、各スキルから参照する。

## 構成

```
skills/
├── slack/
│   └── scripts/           # 共有スクリプト
│       ├── common.js
│       ├── cache.js
│       ├── channels.js
│       ├── users.js
│       ├── history.js
│       ├── thread.js
│       └── search.js
├── slack-channels/
│   └── SKILL.md           # /slack-channels
├── slack-users/
│   └── SKILL.md           # /slack-users
├── slack-history/
│   └── SKILL.md           # /slack-history
├── slack-thread/
│   └── SKILL.md           # /slack-thread
└── slack-search/
    └── SKILL.md           # /slack-search
```

## 各スキルの SKILL.md

### slack-channels
```yaml
---
name: slack-channels
description: "Slack のパブリックチャンネル一覧を取得。Triggers on: /slack-channels, 'チャンネル一覧', 'slackのチャンネル'"
user-invocable: true
allowed-tools:
  - Bash
  - Read
---
scripts/channels.js を引数なしで実行する。
スクリプトのパスはリテラルでフルパスを書くこと（変数展開は使わない）。
```

### slack-users
```yaml
---
name: slack-users
description: "Slack のユーザー・グループ一覧を取得してキャッシュ更新。Triggers on: /slack-users, 'ユーザー一覧', 'ユーザーキャッシュ'"
user-invocable: true
allowed-tools:
  - Bash
  - Read
---
scripts/users.js を引数なしで実行する。
ユーザーグループのキャッシュも同時に更新される。
```

### slack-history
```yaml
---
name: slack-history
description: "Slack チャンネルのメッセージ履歴を取得。Triggers on: /slack-history, 'slackのメッセージ', 'チャンネルの会話', 'slackの履歴', '投稿を確認'"
user-invocable: true
arguments: "<channel> [limit]"
allowed-tools:
  - Bash
  - Read
  - Agent
---
scripts/history.js をチャンネル名orIDと件数で実行する。
キャッシュ未取得時はユーザーに確認の上 slack-channels と slack-users を先に実行する。
出力には ts とメッセージ件数を必ず含めること。
```

### slack-thread
```yaml
---
name: slack-thread
description: "Slack スレッドのメッセージを取得。Triggers on: /slack-thread, 'スレッド', 'スレッドを見せて'"
user-invocable: true
arguments: "<channel> <ts> or <URL>"
allowed-tools:
  - Bash
  - Read
---
scripts/thread.js をチャンネル+ts または Slack URL で実行する。
出力には ts を必ず含めること。
```

### slack-search
```yaml
---
name: slack-search
description: "Slack メッセージをキーワード検索。Triggers on: /slack-search, 'メッセージ検索', 'slack検索', 'slackで検索'"
user-invocable: true
arguments: "<keyword> [count]"
allowed-tools:
  - Bash
  - Read
---
scripts/search.js をキーワードと件数で実行する。
出力には ts とチャンネル名を必ず含めること。
```

## 旧スキル

`skills/slack/SKILL.md` は削除する。
`skills/slack/` ディレクトリは scripts の共有ディレクトリとして残す。

## 依存関係

curl 廃止計画（2026-04-17T1300-refactor-scripts-no-curl.md）を先に実施する。
スクリプトが API を直接叩くようになった後でスキルを分割する。

## 検証

1. `/slack-channels` でチャンネル一覧が取得できること
2. `/slack-history general` でメッセージが表示されること
3. `/slack-thread general <ts>` でスレッドが取得できること
4. `/slack-search keyword` で検索結果が表示されること
5. 自然言語（「generalのメッセージ見せて」）でスキルがトリガーされること
6. 補完が効くこと（`/slack-` で候補が出る）
