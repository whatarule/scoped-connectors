# スキル分割計画

## Context

現在は単一スキル `/slack` でサブコマンド分岐しているが、SKILL.md が長い。
curl 廃止でスクリプトに全責任を移したので、SKILL.md はスクリプト呼び出しだけになっている。
分割すれば各 SKILL.md は数行で済み、`/slack-` の部分一致で補完も効く。

## 決定事項

1. スキル名は `slack-` プレフィックス付き（プラグインスキル補完バグの回避策、plans/KNOWN_ISSUES.md #1）
2. 旧 `skills/slack/SKILL.md` は削除
3. スクリプトは `skills/scripts/` に移動（`skills/slack/scripts/` から）
4. 各 SKILL.md はフルパスリテラルでスクリプトを呼ぶ指示のみ
5. 変数展開は使わない

## 構成

```
skills/
├── scripts/               # 共有スクリプト（旧 skills/slack/scripts/）
│   ├── common.js
│   ├── cache.js
│   ├── channels.js
│   ├── users.js
│   ├── history.js
│   ├── thread.js
│   ├── search.js
│   └── test/
│       ├── common.test.js
│       └── thread.test.js
├── slack-channels/
│   └── SKILL.md
├── slack-users/
│   └── SKILL.md
├── slack-history/
│   └── SKILL.md
├── slack-thread/
│   └── SKILL.md
└── slack-search/
    └── SKILL.md
```

## 各スキルの SKILL.md

### slack-channels

```yaml
---
name: slack-channels
description: "Slack のパブリックチャンネル一覧を取得してキャッシュ更新。Triggers on: /slack-channels, 'チャンネル一覧', 'slackのチャンネル'"
user-invocable: true
allowed-tools:
  - Bash
  - Read
---
```

スクリプトをフルパスリテラルで実行する。変数展開は使わない。

```bash
node /path/to/skills/scripts/channels.js
```

### slack-users

```yaml
---
name: slack-users
description: "Slack のユーザー・グループ一覧を取得してキャッシュ更新。Triggers on: /slack-users, 'ユーザー一覧', 'ユーザーキャッシュ'"
user-invocable: true
allowed-tools:
  - Bash
  - Read
---
```

```bash
node /path/to/skills/scripts/users.js
```

### slack-history

```yaml
---
name: slack-history
description: "Slack チャンネルのメッセージ履歴を取得。Triggers on: /slack-history, 'slackのメッセージ', 'slackの履歴', 'チャンネルの会話', '投稿を確認', 'slackを確認'"
user-invocable: true
arguments: "<channel> [limit]"
allowed-tools:
  - Bash
  - Read
  - Agent
---
```

```bash
node /path/to/skills/scripts/history.js <channel> [limit]
```

出力には ts とメッセージ件数を必ず含めること。

### slack-thread

```yaml
---
name: slack-thread
description: "Slack スレッドのメッセージを取得。Triggers on: /slack-thread, 'スレッド', 'スレッドを見せて'"
user-invocable: true
arguments: "<channel> <ts> or <URL>"
allowed-tools:
  - Bash
  - Read
---
```

```bash
node /path/to/skills/scripts/thread.js <channel> <ts>
node /path/to/skills/scripts/thread.js <slack URL>
```

出力には ts を必ず含めること。

### slack-search

```yaml
---
name: slack-search
description: "Slack メッセージをキーワード検索。Triggers on: /slack-search, 'メッセージ検索', 'slack検索', 'slackで検索'"
user-invocable: true
arguments: "<keyword> [count]"
allowed-tools:
  - Bash
  - Read
---
```

```bash
node /path/to/skills/scripts/search.js <keyword> [count]
```

出力には ts とチャンネル名を必ず含めること。

## 実装順序

1. `skills/slack/scripts/` を `skills/scripts/` に移動
2. `skills/slack/SKILL.md` を削除
3. 5つの SKILL.md を作成
4. cache.js のキャッシュパス（`__dirname` 基準）を更新
5. テスト実行
6. 動作確認

## 検証

1. `/slack:slack-channels` でチャンネル一覧が取得できること
2. `/slack:slack-history general` でメッセージが表示されること
3. `/slack:slack-thread general <ts>` でスレッドが取得できること
4. `/slack:slack-search keyword` で検索結果が表示されること
5. `/slack-` の部分一致で補完候補が出ること
6. 自然言語トリガーで手動許可なしに実行できること
7. テストが通ること
